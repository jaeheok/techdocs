
기본적으로 마이크로 소프트의 IIS상의 백업 / 복구는 기존 서버의 IIS의 삭제 설치 후의
복구를 보장 합니다.
 
해당 메타베이스 파일로는 완전한 OS의 재 설치후의 IIS 복원을 기대 하기 어렵습니다.
 
이에 MetaEdit2.2를 설치 하여 text 형식의 파일로 정보를 추출해 옮기는 유틸리티를
 
활용 하실 수 있습니다.
 
해당 내용을 간략하게 txt로 작성하여 보내드리오니 확인을 부탁드립니다.
 
감사합니다.

Metabase Editor 2.2 설치 

http://support.microsoft.com/kb/301386/ko 에서 Metaedit 2.2를 다운 받습니다.

여느 설치 프로그램과 마찬가지로 기본 설치 진행을 합니다.

영문 기준이기에 adminitrative Tools 로 기본 프로그램 그룹이 등록 됩니다.
 
1. Metabase Editor를 실행 한다.

LM - MSFTPSVC 는 FTP 서버를 가르킨다.
   - W3SVC 는 WEB 서버를 가르킨다.
   
2. 해당 서버(MSFTPSVC or W3SVC or 각 서버들의 하단에 있는 폴더들)들을 선택하고
   상단 메뉴의 Metabase 에서 Export Text 를 선택한다.

※ W3SVC 의 아래의 숫자 폴더들이 각각 사용자들이 생성한 사이트들 입니다.

   그럼 각 서버의 정보가 txt로 출력되어 저장된다.

3. 이 파일을 복원 하고자 하는 IIS서버로 가져와 상단 메뉴의 Metabase Import Text를
   실행 하고 해당 txt를 가져오면 됩니다.

※기본 적으로 metabase 2.2는 IIS 4.0과 5.0까지 지원을 보장합니다.
※기본 웹사이트 , 기본 FTP 사이트 , 기본 관리 사이트등 최초 설치시에 되어 있던
  IIS 기본 사이트들이 복구시에 이름 값이 손상되는 경우가 있습니다.(IP로 이름이 바뀐다던지)
  이 부분은 그냥 수동으로 이름을 원래로 변경 해주시면 됩니다.

4. 완료를 하면 재부팅을 하고 서비스를 구동하여 확인을 합니다.

5. 문제가 있다고 판단되면 IIS를 구성요소 추가 삭제에서 삭제하고 IIS를 다시 설치 하시면 됩니다.
