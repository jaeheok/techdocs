 
기존에 보내 드렸던 addusers.exe는 다음과 같습니다.
일단 계정 명과 해당 계정이 속해 있던 그룹만 복구 된다는 문제점이 있습니다.
 
따라서 최소한 거의 완벽하게 복구 하려면 AD가 아니라는 기준하에
 
내컴퓨터 -> 관리 -> 로컬 사용자 및 그룹 -> 사용자 및 그룹에서 오른쪽 클릭하여
 
그룹을 내보내기 하여 참고할 문서를 저장하셔서 참조 하셔야 하며
 
계정이 복구 되도 해당 계정이 올바르게 그룹에 속해 있나 그룹 생성 부분과
 
내용을 참고 하셔야 합니다.
 
일단 이번에 첨부 해 드리는 유틸리티는
 
www.systemtools.com에서 프리유틸로 제공하는 copypwd란 프로그램입니다.
 
사용법은 단순합니다.
 
다만 로컬상에서 실행 해야 하는 점은 감안 해야 하며 부팅되는 시스템에서만
 
효과가 있습니다.
 
복구 프로그램을 아무데나 압축을 푸신뒤 해당 폴더로 찾아가 커맨드 명령을 수행합니다.
 
copypwd dump > copypwd.txt
 
하면 계정과 패스워드의 정보가 txt로 저장되나 암호화된 내용으로 저장되오니 패스워드를
알수는 없습니다.
 
복구는 복구하고자 하는 시스템에 해당 계정이 모두 존재 한다면
 
copypwd set
 
이라고 명령을 내려 주시면 됩니다. 유틸이 단순하게 되어 있어 txt파일명을 무조건 copypwd.txt로
읽습니다.
 
그러면 정상적으로 기존 패스워드를 그대로 가져 오실수 있습니다.
 
간단하게 테스트를 해보시려면 계정을 몇개 추가 하시고 해당 계정에 패스워드를 1차로
지정 하신뒤 백업을 해보시고 2차로 패스워드를 변경 하신 뒤 복구를 시도 해 보시면
 
터미널 연결시에 1차로 지정된 패스워드로 복구 된 것으만 로그인이 가능 한 것을 확인
하실 수 있습니다.(제가 확인을 마친 테스트 OS환경은 windows2003 Enterprise Edition입니다.)
 
이상입니다.
